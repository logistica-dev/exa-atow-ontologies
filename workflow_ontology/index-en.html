Ontology Specification Draft

Abstract

The Exa-AToW Workflow Ontology provides a semantic framework for describing, integrating, and deploying heterogeneous workflows across scientific computing facilities. It extends and aligns with established W3C standards—PROV-O, wfdesc, wfprov, and PAV—while introducing domain-specific concepts for workflow engines, execution environments, and cross-system interoperability. Designed for the Exa-AToW project under PEPR Numpex, this ontology enables transparent workflow translation between systems such as CWL, NiFi, and Python, and facilitates structured communication between workflow components and external services. By unifying structural, provenance, and deployment semantics, Exa-AToW supports reproducible, portable, and interoperable workflows for exascale scientific applications. test

Introduction back to ToC

Scientific workflows are increasingly executed across distributed and heterogeneous computing environments, often involving multiple workflow management systems (WMS) such as CWL, NiFi, and Python-based orchestrators. However, the lack of semantic interoperability between these systems hinders the portability, reproducibility, and integration of workflows at scale. The Exa-AToW Workflow Ontology addresses this challenge by providing a unifying semantic model that describes
workflows
, their
execution environments
,
data exchanges
, and
provenance
across diverse WMS implementations. Built upon established standards—PROV-O for provenance, wf4ever (wfdesc/wfprov) for workflow descriptions and executions, and PAV for versioning—the ontology introduces additional constructs to represent workflow engines, deployment contexts, and cross-system links. This integration enables seamless translation between workflow systems and facilitates communication with external services and data management components, supporting automated, cross-facility workflow deployment in exascale scientific computing contexts. This ontology extends and aligns with:
  • PROV-O: for provenance tracking of workflow components and executions.
  • wfdesc: for workflow structure elements
  • wfprov: for workflow execution provenance
  • PAV: for versioning and attribution
Workflow Properties are based on: - Suter, F., et al. (2026). A terminology for scientific workflow systems. FGCS 174, 107974. https://doi.org/10.1016/j.future.2025.107974

Exa-AToW Workflow Ontology: Description back to ToC

Ontology for describing workflows, engines, steps, data, provenance, execution environment and design properties for heterogeneous WMS interoperability. This ontology extends and aligns with: - PROV-O: for provenance tracking (prov:used, prov:wasGeneratedBy, prov:wasAssociatedWith) - wfdesc: for workflow structure (wfdesc:hasInput, wfdesc:hasOutput, wfdesc:hasDataLink) - wfprov: for workflow execution provenance - PAV: for versioning and attribution Exa-AToW adds domain-specific classes for workflow properties, execution environments, and cross-WMS interoperability while maintaining full compatibility with W3C standards.
